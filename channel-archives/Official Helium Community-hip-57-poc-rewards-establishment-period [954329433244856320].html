# HIP 57: PoC Rewards Establishment Period - Author(s): @Joey, @scott, @Starwatcher - Start Date: 2022-03-16 - Category: Technical - Original HIP PR: https://github.com/helium/HIP/pull/375 - Tracking Issue: https://github.com/helium/HIP/issues/376 - Status: In Discussion # Summary In order to combat PoC rewards gaming, the process of ‘spoofing’ or ‘gaming’ a Hotspot to earn disproportionate rewards must be economically unsound. Among a range of solutions, the existing denylist and HIP-40 offer a means to remove Hotspots that don’t provide legitimate coverage. However, there is sufficient time after onboarding a Hotspot where it remains economically viable to spoof a location and game PoC rewards before a Hotspot in question can be confidently identified and added to a denylist. If approved, this proposal would introduce a Hotspot ‘establishment period’ such that the network can better learn about a Hotspot’s deployment immediately following a location assertion. This period would be defined initially as 15 days and 10 successful witnesses, as set by a chain variable. A Hotspot’s PoC rewards and witnesses of its beacons would be throttled to 10% of what would be its normal earnings until both of these conditions are met. Data rewards would be unaffected. As the network improves its ability to understand a Hotspot’s location or as reliance on denylists are reduced, these chain variables could be adjusted. # Summary https://github.com/helium/HIP/blob/main/0057-poc-rewards-establishment-period.md

GPS in miners are often given as a solution which is in fact unviable. GPS spoofing is easy, it’s passive. Active satellites with uplinks instead of just a downlink, like SpaceX’s Starlink can triangulate a geolocation at the point of antenna assertion and issue a private key that will then only unlock the public hash of that location for the hex owner. One doesn’t need an entire starlink node to send and receive low bandwidth data, just an antenna in their miner. The problem with this is that anyone could take a starlink antenna to any random place and geoclaim it, especially if they had unlimited hotspots in the back of a van and began to claim entire regions. The solution to that would be to only allow a hex to be claimed by a single miner within hearing range of an existing hex, and that’s where it stays, so you can’t just claim vast areas of land without part of the network already existing along the boundary of that area, and the boundary won’t exist if it’s not maintained via periodic duplex sat geolocation. The outcome of all of the above is that once a legitimate miner is set up in a specific hex, it’s taken via a non-fungible token, and can only be transferred to a new owner if the existing owner gives permission to transfer it away, and so that geolocation has an intrinsic scarcity value that not only prevents saturation, but also incentivises expansion of coverage.

To address the situation of diminishing rewards as the network grows (and prior to it becoming incentivised as a future data transfer node), the existing halving structure of HNT rewards must rescale proportionally and in a deflationary fashion, such that a growing number of hotspots does not result in existing hex owner rewards become diluted. Each new miner onboarding should burn the equivalent in the available supply in all rewards issued throughout it’s life. This is achieved in a deflationary HNT asset which burns a percentage of the market cap to assert a new hex, rather than a set fee to encroach upon someone elses. This maintains a cost of entry to mining HNT which is not dependant upon scarcity of hotspot hardware, but rather available hexes. Halving’s would occur instead upon a percentage of geolocated land assertions taken. The current implementation in which a miner onboards to a hyperinflationary HNT structure which sees their efforts to build out the network put into ruin by anyone else asserting mining hardware in their close proximity is fatally flawed. HNT’s current value is a reflection of the cost of entry and delays in miner production, which is rapidly reaching saturation. One can not simply buy a faster more lucrative HNT miner for a competitive edge, and one can not prevent all of their efforts into building and maintaining a spacious and well organised network being ruined by anyone next door who can buy a miner. If Helium don’t create an intrinsic value to a geolocated hotspot, then saturation will be it’s undoing.
